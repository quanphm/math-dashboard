# AGENTS.md

## Quick Reference

### Local Development URLs
- **Frontend:** `https://hoalu.localhost`
- **API:** `https://hoalu.localhost/api`

### Essential Commands

```bash
# Install dependencies
bun install --ignore-scripts

# Start infrastructure (PostgreSQL, Redis, Electric)
bun run docker:up

# Start Caddy reverse proxy (required for local HTTPS)
caddy run

# Start dev servers (API + Dashboard)
bun dev

# Database operations
cd apps/api && bun run db:generate   # Generate migrations after schema changes
cd apps/api && bun run db:migrate    # Run pending migrations
cd apps/api && bun run db:seed       # Seed database

# Code quality
bun run lint        # oxlint
bun run lint:fix    # oxlint --fix
bun run format      # oxfmt --write .
bun run format:check # oxfmt --check .
```

## Critical Constraints

### NEVER
- ❌ Modify `apps/dashboard/src/lib/api-client.ts` - auto-generated from Hono RPC
- ❌ Edit `apps/dashboard/src/routeTree.gen.ts` - generated by TanStack Router
- ❌ Use relative imports like `../../` - use path aliases
- ❌ Omit `.ts`/`.tsx` extensions in imports - Bun requirement
- ❌ Skip migrations after schema changes
- ❌ Create routes without workspace scoping

### ALWAYS
- ✅ Use path aliases: `#api/*` (backend), `@math/ui/*` (frontend)
- ✅ Include file extensions in all imports
- ✅ Run `db:generate` → `db:migrate` after schema changes
- ✅ Validate with Zod before database operations
- ✅ Scope all data by `workspace_id` (multi-tenancy)
- ✅ Use `z.coerce.number()` for Electric SQL numeric fields (sent as strings)

## Code Conventions

### Path Aliases

**Backend** (`apps/api/package.json`):
```json
{
  "imports": {
    "#api/*": "./src/*"
  }
}
```

**Frontend** uses workspace packages:
```json
"@math/ui": "workspace:*"
```

### Import Rules
- ✅ `import { Button } from "@math/ui/components/button.tsx"`
- ❌ `import { Button } from "../../packages/ui/components/button"`

### Zod v4 Patterns

```typescript
// Coerce Electric SQL string numbers
const schema = z.object({
  amount: z.coerce.number(),      // "123.45" → 123.45
  quantity: z.coerce.number().int(),
  date: z.iso.datetime(),         // Zod v4 ISO datetime
  id: z.uuidv7(),                 // Zod v4 UUID v7
});
```

## Tooling Configuration

### Linting & Formatting (oxc)
- **Linter:** oxlint (configured in `.oxlintrc.json`)
- **Formatter:** oxfmt (configured in `.oxfmtrc.json`)
- **VS Code:** OXC extension recommended

Key oxlint rules:
- React/JSX a11y plugins enabled
- TypeScript correctness = error
- `no-explicit-any` = off (allowed)
- Generated files ignored: `routeTree.gen.ts`, `dist/**`, etc.

### Formatter Settings
- Tabs, 2-space width
- Print width: 100
- Trailing commas: all
- Import sorting with TanStack group priority
- Tailwind class sorting for `cn`, `clsx`, `cva`

### TypeScript
- Workspace SDK: `node_modules/typescript/lib`
- Generated files marked read-only in VS Code

## Planned Architecture (Not Yet Implemented)

The following patterns are documented for when apps are initialized:

### API Route Pattern (3 files)
```
apps/api/src/routes/[resource]/
├── index.ts       # Route handlers
├── repository.ts  # Database queries
└── schema.ts      # Zod schemas
```

### Frontend Route Pattern
- File-based routing with TanStack Router
- Routes in `apps/dashboard/src/routes/`
- Route tree auto-generated

### Data Flow
1. Electric SQL syncs to local PGlite
2. TanStack Query for server state
3. Live queries via `useExpenses()` hooks
4. Mutations in `services/mutations.ts`

### Auth Pattern
- Better Auth with workspace plugin
- Workspace-scoped multi-tenancy
- Middleware chain: `workspaceQueryValidator` → `workspaceMember` → handler

## Common Tasks Reference

| Task | Command/Location |
|------|-----------------|
| Add API route | `apps/api/src/routes/[resource]/` (3-file pattern) |
| Add DB table | Edit `apps/api/src/db/schema.ts` → `db:generate` → `db:migrate` |

| Fix type errors | Restart dev servers (RPC/route regeneration) |
| Reset local DB | `bun run docker:down` → delete `pg-data/` → `docker:up` |
